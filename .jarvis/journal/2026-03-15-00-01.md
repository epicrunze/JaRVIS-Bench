## What Worked
- Parallel file creation sped things up significantly — writing 9 files at once.
- Reading scaffold.md first gave complete context for all decisions.
- The incremental approach to setup.sh — write, run, fix, run — caught real issues quickly.

## What Didn't Work
- Initial `((READY++))` arithmetic under `set -euo pipefail` was a subtle bash pitfall. Should have used `$((var + 1))` assignment or wrapper functions from the start.
- Assumed `pip install -e .` would work without a `[build-system]` table in pyproject.toml — modern setuptools requires it.
- Assumed JaRVIS skills were `.md` files without checking the actual directory structure first.

## Lessons Learned
- In bash with `set -e`, never use `((var++))` when var could be 0 — it evaluates to falsy and exits. Use `var=$((var + 1))` or wrapper functions instead.
- Always include `[build-system]` and `[tool.setuptools.packages.find]` in pyproject.toml for projects with non-standard layouts (multiple top-level dirs like vendor/, results/).
- User prefers `uv` over raw `pip` for Python package management.
- Always check actual file/directory structure of vendored dependencies before writing code that depends on them.
